# Plan 14: Multi-Locale Testing

**Priority:** Tier 3 — Lower Impact / Niche
**Personas:** i18n Engineer
**New commands:** `i18n`

---

## Problem

There's no way to run the same flow across multiple locales and compare results. i18n engineers need to detect: missing translations (raw keys showing), RTL layout issues, date/number format inconsistencies, and truncated text from longer translations.

## Design

### Command Interface

```bash
# Run a flow across multiple locales
browse i18n --flow checkout --locales en,fr,de,ja,ar

# Compare specific pages across locales
browse i18n --url https://example.com/pricing --locales en,fr,de

# Check for untranslated strings (raw key patterns)
browse i18n check-keys --url https://example.com --pattern "^[A-Z_]+\.[A-Z_]+"

# RTL layout check
browse i18n rtl-check --url https://example.com --locale ar

# Generate comparison report
browse i18n --flow checkout --locales en,fr,de,ja,ar --report i18n-report.html

# JSON output
browse i18n --flow checkout --locales en,fr --json
```

### Checks Performed Per Locale

#### 1. Missing Translations
- Scan visible text for patterns that look like translation keys:
  - `UPPER_SNAKE.CASE.KEYS`
  - `dot.separated.keys`
  - `{{placeholder}}` syntax (unexpanded)
  - Common i18n library fallback patterns
- Custom pattern via `--pattern` regex

#### 2. RTL Layout Validation (for ar, he, fa, ur)
- Check `<html dir="rtl">` or `direction: rtl` in computed styles
- Screenshot and compare layout against LTR version
- Detect text overflow / truncation via element scroll dimensions
- Check that navigation order is reversed

#### 3. Text Overflow / Truncation
- For each interactive element: compare `scrollWidth` vs `clientWidth`
- German and Finnish text is typically 30-40% longer than English
- Flag elements where translated text overflows its container

#### 4. Date/Number Formatting
- Search page text for date patterns and number patterns
- Verify they match the expected locale format:
  - English: `03/22/2026`, `1,234.56`
  - German: `22.03.2026`, `1.234,56`
  - Japanese: `2026年3月22日`

#### 5. Screenshot Comparison
- Capture screenshot for each locale at same viewport
- Visual diff between base locale (en) and each other locale
- Flag significant layout shifts

### Implementation

**File:** `src/commands/i18n.ts` (~300 lines)
**File:** `src/i18n-engine.ts` (~350 lines)

1. **Locale switching**:
   - Set browser locale via Playwright context options: `locale`, `timezoneId`
   - Requires creating a new context per locale (or reconfiguring existing)
   - Set `Accept-Language` header
   - If app uses URL-based locale (`/en/`, `/fr/`), transform URLs with locale prefix

2. **Per-locale execution**:
   - For each locale: set context, run flow/navigate, run checks, capture screenshot
   - Collect results in structured format

3. **Comparison engine**:
   - Diff text content between locales (flag identical text = possibly untranslated)
   - Diff screenshots using existing `visual-diff.ts`
   - Aggregate findings by severity
